# Lite workflow

1. Read the repository `AGENTS.md`, map, and current git status.
2. State a safe assumption and proceed; ask only when the answer materially changes the result.
3. Trace the smallest relevant surface. For bugs, identify the cause before editing.
4. Create or update a short plan when the work has more than one step.
5. Implement the smallest complete change. Prefer the root agent; delegate only authorized, independent research or a disjoint file unit.
6. Review the diff and run the repository DoD.
7. Commit, push, or create a PR only when requested. Stage only owned files and preserve user authorship.
8. If the work expands beyond one task window, record status in `plan.md` and escalate to `medium.md`.

After a completed cycle, offer the cleanup kickoff: `Read ~/.codex/cleaning/README.md and run it.`
