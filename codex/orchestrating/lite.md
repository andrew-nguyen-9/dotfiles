# Lite workflow

1. Start Caveman ultra and Ponytail ultra unless the user overrides them; escalate clarity/correctness when compression or minimalism risks the result.
2. Read the repository `AGENTS.md`, map, and current git status.
3. State a safe assumption and proceed; ask only when the answer materially changes the result.
4. Trace the smallest relevant surface. For bugs, identify the cause before editing.
5. Create or update a short plan when the work has more than one step.
6. Implement the smallest complete change. Prefer the root agent; delegate only authorized, independent research or a disjoint file unit.
7. Review the diff and run the repository DoD.
8. Commit, push, or create a PR only when requested. Stage only owned files and preserve user authorship.
9. If the work expands beyond one task window, record status in `plan.md` and escalate to `medium.md`.

After a completed cycle, offer the cleanup kickoff: `Read ~/.codex/cleaning/README.md and run it.`
